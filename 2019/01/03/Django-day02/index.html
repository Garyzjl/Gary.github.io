Django虚拟环境构建

  1. 安装virtualenv
    在cmd终端控制台，输入：pip install virtualenv
  2. 创建虚拟环境
    可创建一个专门放置虚拟环境的文件夹env（个人习惯），在cmd来到该文件夹下，输入：virtualenv –no-site-packages -p pyhon版本路径\python.exe name
    注：1.–no-site-packages:表示创建的虚拟环境为纯净的环境，不安装有其他的库。
    2.-p：表示虚拟环境中的python版本。如果只安装有一个python版本，-p pyhon版本路径\python.exe可以省略不写。直接输入：virtualenv –no-site-packages name
    3.name 为创建虚拟环境配置文件夹名称
  3. 激活虚拟环境
    输入：cd name(创建时的名称) Scripts 来到该路径下
    输入：activate 激活虚拟环境
    退出虚拟环境：deactivate
  4. 安装Django库
    输入：pip install django
    成功安装后，即环境创建成功！

    创建Django项目

  5. 创建项目
    可创建一个目录用于单独存放Django项目。在cmd控制台来到所创建的目录下，
    输入：django-admin startproject name(项目名称)
  6. 打开项目,并配置
    2.1 用pycharm打开创建好的项目，在file-settings-Project: name(项目名称)-Project Interpreter中的Project Interpreter选中所创建好的虚拟环境的Scripts下的python.exe。
    2.2 在pycharm 主界面的右上方，创建的name(项目名称)处单击, 点Edit-Configurations,来到Run/Debug configurations界面；点加号，修改Name为创建项目的name(项目名称)；Scripts path选择创建项目中manage.py文件；
    2.3.Parameters中输入：runserver 端口。端口默认为8000，可输入自定义设置的端口号。
    2.4 点击OK，完成
  7. 创建应用
    在Terminal 中输入：python manage.py startapp app(应用名称)

    启动Django项目

    在pycharm的Terminal中输入：python manage.py runserver
    注：要在创建虚拟环境的下输入

    数据库准备

  8. settings配置
    在settings中的DATABASES中配置参数
    ENGING: ‘django.db.backends.mysql’ 根据所用数据类型设置，例如本人使用的是mysql
    USER: 访问用户
    PASSWORD: 密码
    HOST: 访问数据库的地址
    PORT: 访问数据库的端口
    NAME: 数据库名称
    TIME_ZONE: 所在时区（默认为UTC，可以不设置）
  9. 安装、配置pymysql
    在Terminal 中输入: pip install pymysql
    在工程目录下init.py文件中定义：pymysql.install_as_MySQLdb()
  10. 迁移
    第一次迁移：Terminal中输入：python manage.py migrate
    除开第一次迁移：
    生成文件迁移
    Terminal中输入：python manage.py makemigrations
    执行迁移文件
    Terminal中输入：python manage.py migrate
  11. 创建管理员账号
    Terminal中输入：python manage.py createsuperuser

  1. 字段类型
    CharField: 字符串类型
    IntegerField: 整型
    ImageField：字符串类型，用于图片
    BooleanField: 布尔，True 或者 False
    DateTimeField: 日期，年月日时分秒
    DateField: 年月日
    TextField: 存储长文本内容，areatext标签
    FloatField: 浮点类型
    DecimalField: 浮点类型，限制最大长度和小数点后的长度
  2. 约束
    unique：是否唯一
    default：默认值
    null：是否可以为空
    primary_key: 主键
    auto_now_add: 创建时，默认赋值为当前时间
    auto_now: 创建或修改时，默认值为当前时间
    max_length: 最大长度
    related_name: 反向查询名称，即别名

on_delete: 删除的约束条件：
models.CASCADE 删除主表，从表也会被删
models.PROTECT 不能删除主表
models.SET_NULL 删除主表，从表的关联字段设置为空
